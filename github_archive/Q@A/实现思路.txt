## 逐步评估

### 第1-2步：MusicGen + 端到端微调
- 难度：中等
- 单独价值：**不够发一区**，这只是工程实现，创新性不够
- 作用：是整个系统的基础

### 第3步：theta调制约束
- 难度：低（信号处理层面）
- 但这是**论文的核心创新点**
- 关键在于**怎么做**：

| 做法 | 难度 | 发表层次 |
|-----|------|---------|
| 生成后简单叠加theta包络（后处理） | 低 | 三四区 |
| 把theta调制参数作为MusicGen的训练条件（端到端） | 中 | 二区 |
| theta调制 + 个人偏好建模 + 闭环反馈 | 高 | 一区（TAFFC） |

### 第4步：用户实验 + EEG验证
- 难度：**这步最难**，不是技术难，是执行难
- 需要：被试招募、EEG设备、伦理审批、数据分析
- 但这步是**发一区的必要条件**——没有真人实验数据，TAFFC不会收

## 层次一：生成后叠加theta包络（三四区）

**原理**：先用MusicGen生成音乐，然后在音频波形上叠加4-8Hz的振幅调制

**需要学的**：

| 知识点 | 具体内容 | 学习时间 |
|-------|---------|---------|
| 信号处理基础 | 什么是包络、调幅(AM)、频率调制 | 2天 |
| scipy.signal | 生成正弦波、低通滤波、包络提取 | 1天 |
| theta节律 | 4-8Hz是什么、为什么和情绪/注意力有关 | 1天（读论文） |

**代码上就是**：
```
生成的音乐波形 × (1 + 0.3 × sin(2π × 6Hz × t))
```
一行代码就能实现，所以说难度低。

---

## 层次二：theta作为MusicGen训练条件（二区）

**原理**：不是后处理叠加，而是让MusicGen**学会生成自带theta调制特征的音乐**

**需要学的**：

| 知识点 | 具体内容 | 学习时间 |
|-------|---------|---------|
| **MusicGen微调** | 你已经在做的端到端微调（train_e2e.py） | 已掌握框架 |
| **音频特征提取** | 从音频中提取theta频段能量、频谱特征 | 1周 |
| **条件编码设计** | 怎么把theta参数编进MusicGen的条件空间 | 1周 |
| **损失函数设计** | 除了token预测loss，加一个theta功率约束loss | 3天 |
| librosa | 音频分析库，计算频谱、MFCC、节奏等 | 3天 |

**核心思路**：
```
训练数据准备：
  对每首EMOPIA音乐，计算它的theta频段能量 → 得到theta_score
  训练对变成：(情绪标签, theta_score) → 音乐token

训练时：
  条件输入 = [情绪嵌入, theta参数嵌入]  ← 两个条件拼接
  MusicGen学会：不同theta参数 → 不同调制程度的音乐

推理时：
  输入(抑郁, theta=高) → 生成强theta调制的治疗音乐
```

## 层次三：theta + 个人偏好 + 闭环（一区TAFFC）

**原理**：不只生成治疗音乐，还要根据用户实时EEG反馈动态调整

**需要学的**（在层次二基础上额外学）：

### A. 个人偏好建模
| 知识点 | 具体内容 | 学习时间 |
|-------|---------|---------|
| 推荐系统基础 | 用户embedding、协同过滤的概念 | 3天 |
| 偏好编码 | 把用户的音乐偏好（喜欢的风格、节奏）编码成向量 | 1周 |

Cell Reports论文的核心发现就是"个人喜好决定疗效"，所以偏好建模是冲一区的必要创新点。

### B. EEG实时分析
| 知识点 | 具体内容 | 学习时间 |
|-------|---------|---------|
| **MNE-Python** | EEG数据处理标准库，读取、滤波、频谱分析 | 1-2周 |
| theta功率计算 | 对颞区电极提取4-8Hz频段功率 | 包含在MNE学习中 |
| wPLI计算 | 加权相位滞后指数，衡量音乐和EEG的同步性 | 3天 |
| 实时流处理 | 用LSL（Lab Streaming Layer）做实时EEG采集 | 1周 |

### C. 闭环反馈系统
| 知识点 | 具体内容 | 学习时间 |
|-------|---------|---------|
| 控制论基础 | 什么是闭环控制、PID的概念 | 2天（了解即可） |
| 反馈策略 | 每30秒检测EEG → theta低了就加强调制 → 重新生成 | 1周 |
| 系统集成 | 把EEG采集、分析、MusicGen生成串成实时管道 | 2周 |

**闭环的核心逻辑**：
```
循环开始：
  1. 播放生成的音乐（30秒片段）
  2. 同时记录EEG
  3. 计算颞区theta功率
  4. 如果theta低 → 调高theta调制参数 → 重新生成下一个片段
  5. 如果theta高 → 维持当前参数
  回到1
```

### D. 用户实验设计
| 知识点 | 具体内容 | 学习时间 |
|-------|---------|---------|
| 实验设计 | 组内/组间设计、对照组、随机化 | 3天 |
| SAM量表 | 情绪自评（效价、唤醒度）的标准工具 | 1天 |
| 统计分析 | t检验、ANOVA、效应量，用scipy.stats | 3天 |
| 论文写作 | 方法/结果/讨论的写法 | 跟导师学 |

## 系统输入输出

```
┌─────────────────────────────────────────────┐
│                  你的系统                      │
│                                              │
│  输入1: EEG情绪特征(同学的SNN输出)             │
│    → "这个人现在是抑郁/焦虑/平静"              │
│    → 决定生成什么条件的音乐                    │
│                                              │
│  输入2: theta调制参数(你自己设定)               │
│    → "治疗性音乐需要多强的theta调制"           │
│                                              │
│         ↓ 两个条件一起喂给MusicGen              │
│                                              │
│  输出: 治疗性音乐                              │
│                                              │
└─────────────────────────────────────────────┘
                    ↓
             用户听这个音乐
                    ↓
         同时记录EEG → 看theta功率变了没（评估用）
```

## 简单说

| | EEG情绪特征 | EEG theta功率 |
|---|---|---|
| 来源 | 同学的SNN模型提取 | 你自己用MNE-Python算 |
| 用在哪 | 系统**输入**（生成条件） | 系统**输出评估**（验证疗效） |
| 什么时候用 | 生成音乐之前 | 听音乐的时候/之后 |
| 谁负责 | 同学 | 你 |

**所以你的同学那部分工作没有白费**，情绪特征输入还在。只是你额外加了theta调制条件和EEG评估指标，让系统从"情绪匹配"升级成了"神经靶向治疗"。

这就是完整的故事：**同学识别情绪 → 你根据情绪+theta参数生成治疗音乐 → 用EEG验证效果**。两人合作，一篇论文。
